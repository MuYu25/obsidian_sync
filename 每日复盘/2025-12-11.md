## **今天完成了什么？值得肯定的地方是什么？**（目的：建立信心，看到进步，而不是只盯着没做完的事）
- 今天解决了一个临时的`bug`,是提分计划中设置最小值的`ecpm`值得筛选还是会展示`ecpm`值为null的用户。因为`join`时不设置可以接受`null`就会出现为`null`的值不会进行展示。
```go
  // 新代码
hasMinEcpm := false
if ecpm_min := req.Wheres.Get("ecpm_min"); !ecpm_min.IsEmpty() {
	minEcpm = ecpm_min.Int()
	hasMinEcpm = true
}

var ecpmCond string
if hasMinEcpm {
	ecpmCond = "t.three_avg_ecpm between ? and ?"
} else {
	ecpmCond = "(t.three_avg_ecpm between ? and ? OR t.three_avg_ecpm is null)"
}
// 执行查询
```
- 还一点做的比较好，就是今天考虑了所有情况，什么情况需要不能够设置`OR ecpm is null`，一共有四种情况:
	- 有最小值限制，有最大值限制，不能够添加`or`条件.
	- 有最小值限制，无最大值限制，不能够添加`or`条件.
	- 无最小值限制，有最大值限制，能够添加`or`条件.
	- 无最小值限制，无最大值限制，能够添加`or`条件.
	那么存在最小值限制时，就不能够添加`or`条件了。
